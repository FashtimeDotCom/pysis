Installing¶

For Users¶

This software is not yet ready for production use.

For Developers¶

Note

In the following instructions, I am assuming that your OS is Ubuntu 10.04 or higher.

  1. Install Python and Mercurial

    sudo apt-get install python mercurial

  2. Install MySQL or PostgreSQL database along with headers.

    sudo apt-get install mysql-client mysql-server libmysqlclient-dev

or

sudo apt-get install postgresql-9.0  postgresql-server-dev-9.0

  3. Install nginx webserver.

    sudo apt-get install nginx

  4. Install supervisor.

    sudo apt-get install supervisor

  5. Install pip.

    sudo easy_install setuptools pip

  6. Install virtualenv.

    sudo pip install virtualenv virtualenvwrapper

  7. Create directories for this project.

    sudo mkdir /virtualenvs
sudo mkdir /projects

sudo chmod 777 /virtualenvs
sudo chmod 777 /projects

mkdir /virtualenvs/download_cache

  8. Add the following to your ~/.bashrc

    export WORKON_HOME=/virtualenvs
export PIP_DOWNLOAD_CACHE=/virtualenvs/download_cache
source /usr/local/bin/virtualenvwrapper.sh

  9. Refresh your session to make the above changes visible.

    source ~/.bashrc

  10. Create virtualenv.

    mkvirtualenv --no-site-packages pysis

  11. Activate virtualenv.

    workon pysis

  12. Note

    From now on, I am assuming that you are inside the virtualenv. A Quick test : when you run “which python” and it should show “/virtualenvs/pysis/bin/python”

  13. Clone pysis repository.

    hg clone http://bitbucket.org/sramana/pysis /projects/pysis

  14. Add /projects to python path.

    add2virtualenv /projects

  15. Install requirements. This will take some time.

    pip install -r /projects/pysis/requirements/requirements.txt

  16. Add a shortcut to manage the project. Edit your ~/.bashrc and add this line.

    source /projects/pysis/scripts/shortcuts.sh

  17. Refresh your session to make the above changes visible.

    source ~/.bashrc

  18. Edit /projects/pysis/settings/production_settings.py and provide your database credentials under DATABASES.

  19. Create tables.

    django-admin.py syncdb
django-admin.py migrate core

  20. Add the nginx configuration.

    sudo touch /etc/nginx/common_server_options.nginx.conf
sudo ln -s /projects/pysis/deploy/pysis_nginx.conf /etc/nginx/sites-enabled/
sudo ln -s /projects/pysis/media /var/www/pysis_media

  21. Restart nginx.

    sudo service nginx restart

  22. Add this project to supervisor. Edit /etc/supervisor/supervisord.conf and add the following lines to the bottom.

    [program:pysis]
command=/virtualenvs/pysis/bin/python /projects/pysis/deploy/pysis_fcgi.py
user=www-data

  23. Reload supervisor.

    sudo supervisorctl reload

  24. Run “pysis” and make sure that no errors are reported.

  25. Add “127.0.1.1 pysis.localhost” to /etc/hosts.

  26. Now point your browser to http://pysis.localhost and enjoy.
